异常

什么是异常？java代码在运行时出现的问题就是异常，在java中 ，把异常信息封装成一个类，当出现问题是，就会创建异常对象相关的信息（如异常的位置，原因等）

Throwable类 Throwable位于java.lang包下 ，是 Java 语言中所有错误（Error）或异常(Exception)的超类

错误（Error）相当于癌症，所以当程序报出Error时候，只能去修改代码

异常(Exception)相当于感冒，我们将异常处理掉，程序可以继续运行

异常中的关键字
throw 在方法内部，抛出异常的对象，在throw后面，必须写出new 对象，必须是异常的对象，必须是Exception或者子类
throws用于方法的声明上，表明方法，可能出现异常，请调用者处理，throws后面必须写异常类的类名

*异常中的处理方式

Blockquote try{被检测的代码
可能出现异常的代码
}catch(异常类名 变量){
异常的处理方式
循环 判断 调用方法 变量
}finally{
必须要执行的代码}

支持多catch处理，但是越高级父类越写在后面
finally意思是无论有没有异常都要执行其中的代码

异常的分类
编译异常:调用抛出的方法，不处理编译失败
运行时异常：抛出的异常的RuntimeException类，或者他的子类

运行时异常的特点：方法的声明上，不需要throws句。

方法重写时候异常的处理
父类的方法，如果抛出异常，子类重写后，可以不抛出异常，也可以抛出异常，但是，如果子类要抛，抛出的异常不能大于（只的是继承关系）父类的异常

Throwable类中的方法
void printStackTrace() 将异常信息追踪到标准的错误流，JVM默认调用这个方法

自定义异常
继承Exception,或者继承RuntimeException
大部分是继承RuntimeException
